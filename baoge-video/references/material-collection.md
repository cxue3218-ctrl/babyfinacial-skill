# 素材类型优先级与采集流程

## 素材类型优先级

1. **平台实拍素材**（B站/YouTube/抖音） — 先找抖音→B站→YouTube，搜索→预筛选→下载→截帧验收
2. **素材网站**（Pexels / Pixabay / Mixkit） — 无水印直接可用，作为平台实拍的补充
3. **AI生成视频** — 优先尝试AI生成动态视频；若模型不具备能力，备选AI图片+FFmpeg运镜
4. **数据可视化动画** — Pillow/matplotlib生成的动画（柱状图、环形图、数字递增等）
5. **禁止**：PPT截图、K线截图、带水印的任何素材

## 素材占比方案（30句脚本参考）

| 素材类型 | 段数 | 占比 | 说明 |
|----------|------|------|------|
| 平台实拍视频 | 10 | 30% | 抖音/B站/YouTube，搜索→预筛选→下载→截帧验收 |
| AI生成视频 | 11 | 35% | 含8段数据可视化动画 + 3段AI生成动态视频 |
| 素材网站 | 9 | 30% | Pexels / Pixabay / Mixkit |
| 合计 | 30 | 100% | |

> 占比可根据脚本实际内容调整，但平台实拍+素材网站合计不低于50%，AI生成+数据可视化不高于50%。

**AI降级规则**：当AI视频生成不可用（只能用AI图片+FFmpeg运镜时），AI生成段数减半（35%→约18%），减少的段数改用素材网站素材填充（30%→约47%）。

## 素材横屏优先原则（核心规则）

**所有平台实拍素材和素材网站素材，优先截取横屏（16:9）画面。**

原因：竖版视频的素材净区是 1080×768（接近 4:3），横屏素材裁剪到 1080×768 时损失小、构图完整；竖屏素材（9:16）裁剪到 1080×768 需要裁掉上下大量内容，容易丢失主体信息。

| 素材方向 | 原始比例 | 裁剪到1080×768 | 效果 |
|----------|----------|----------------|------|
| 横屏 ✅ | 16:9 (1920×1080) | 上下加黑边或轻微裁剪 | 构图完整，主体居中 |
| 竖屏 ❌ | 9:16 (1080×1920) | 需裁掉上下各576px | 损失大量画面 |

**搜索时优先筛选横屏视频**：
- 抖音/B站/YouTube 搜索时，优先选择横屏拍摄的视频（纪录片、航拍、工厂实拍等）
- 排除竖屏拍摄的短视频、口播类、手机拍摄类
- Pexels/Pixabay 搜索时，在筛选条件中选择 Landscape / Horizontal 方向

**例外情况**：当横屏素材实在搜不到，且该镜内容必须用实拍画面时，才退而求其次用竖屏素材，但需额外裁剪处理确保主体不丢失。

## 素材时长要求（核心规则）

**截取素材必须先取10-30秒的连续片段，再按TTS时长裁剪：**

1. 从源视频截取一段 **10-30秒** 的连续流畅画面
2. 从这段素材中按TTS实际时长（3-8秒）做最终裁剪
3. **禁止**直接截取只有3秒的片段（画面来不及展开，生硬）
4. **禁止**使用截屏图片拼接成一卡一卡的伪视频
5. 素材画面必须**丝滑流畅**：有自然运动感，不能是静止画面或跳帧

```bash
# 正确：先截取15秒流畅片段，再按TTS时长裁剪
ffmpeg -i source.mp4 -ss 00:01:20 -t 15 -an raw_clip.mp4
ffmpeg -i raw_clip.mp4 -ss 0 -t 5 -vf "crop=...,scale=1080:768" -an final_clip.mp4

# ❌ 错误：直接截3秒
# ffmpeg -i source.mp4 -ss 00:01:20 -t 3 output.mp4  ← 禁止
```

## 同一长视频多段截取规则

同一个长视频可以在不同进度条截取多个不同画面用于不同分镜，但：
- 每个截取段的画面内容必须不同
- 同一个长视频最多截2-3段不同进度
- 每段仍需截取10-30秒流畅片段再裁剪
- **绝不重复用同一几秒画面贴到多个分镜**

## 平台素材采集详细流程

### 总流程

```
确定关键词 → 打开平台搜索 → 模拟人工浏览预筛选 → 下载 → 截取10-30秒片段 → 清洗裁剪 → 验收
```

### 平台优先级

**按关键词地域动态调整**，详见 `references/keyword-rules.md` 第六节

### 抖音素材采集流程

1. 用Playwright打开 `https://www.douyin.com/search/{关键词}`
2. 在搜索结果页浏览视频列表，**模拟人工操作**：
   - 逐个hover视频缩略图，查看预览画面
   - 判断画面是否无人脸、无字幕、无水印、无字体、与文案匹配
   - 排除明显不合适的（UP主口播类、幻灯片类）
3. 点击进入候选视频，实际播放5-10秒确认画面内容
4. **获取视频真实URL**（二选一）：

   **方法A：转发复制链接 + evaluate提取（推荐）**
   - 在视频播放页点击右侧"转发/分享"按钮
   - 在弹出的分享面板点击"复制链接"
   - 用PowerShell `Get-Clipboard` 读取剪贴板，提取短链接（如 `https://v.douyin.com/xxxxx/`）
   - 用Playwright `browser_navigate` 打开短链接，跳转到视频详情页
   - 用 `browser_evaluate` 执行JS提取video标签的 `currentSrc`：
   ```javascript
   () => {
     const videos = document.querySelectorAll('video');
     const srcs = [];
     videos.forEach(v => srcs.push({src: v.src, currentSrc: v.currentSrc}));
     return JSON.stringify(srcs);
   }
   ```
   - 从返回JSON中取 `currentSrc` 字段（douyinvod.com域名的视频直链）
   - 用ffmpeg直接下载

   **方法B：yt-dlp下载（备选，需cookies）**
   - 先用Playwright获取抖音cookies，转为Netscape格式
   - yt-dlp可能报 "Fresh cookies needed" 错误，此时切换到方法A

5. 用FFmpeg截取10-30秒流畅片段，再按TTS时长裁剪到1080×768

### B站素材采集流程

1. 用Playwright打开 `https://search.bilibili.com/all?keyword={关键词}`
2. 模拟人工浏览搜索结果，优先选择纪录片/工厂实拍/航拍类
3. 点击进入候选视频，处理B站cookies
4. 实际播放视频5-10秒确认画面干净
5. 用yt-dlp下载（带cookies）
6. 截取10-30秒片段 → 裁剪字幕区 → 缩放1080×768

### YouTube素材采集流程

1. 用Playwright打开 `https://www.youtube.com/results?search_query={关键词}`
2. 模拟人工浏览，排除有人脸口播的频道视频
3. 播放确认画面内容
4. 获取cookies转为Netscape格式
5. 用yt-dlp下载（带cookies），长视频用 `--download-sections` 只下载所需片段
6. 截取10-30秒片段 → 裁剪 → 缩放1080×768
