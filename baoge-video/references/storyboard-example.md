## Search Query 检索词生成公式

系统（Agent）在设计分镜清单的 search_query（检索词）时，绝对禁止使用具体的局部微观动作或极窄词汇（如：焊接芯片、拧螺丝、指纹解锁）。必须遵循以下"宽泛词语，例如行业、资金、K线、大门类、大场景、大空镜"的检索式生成公式，否则退回重写：

```
Search Query (中文或英文) = [大行业/大门类主题词] + [空间/宏观场景载体]
```

### 正确示例

| 检索词 | 公式拆解 |
|--------|----------|
| stock market trading | 金融投资大门类 + 交易场景 |
| stock market bull rally | 金融投资大门类 + 牛市场景 |
| wafer fabrication silicon wafer | 半导体大门类 + 制造场景 |
| 存储芯片 仓库 | 存储半导体大门类 + 仓储场景 |
| power semiconductor equipment | 功率半导体大门类 + 设备场景 |
| MLCC capacitor | 电子元件大门类 + 产品载体 |
| optical fiber production | 光通信大门类 + 生产场景 |
| semiconductor gas chemical | 半导体材料大门类 + 化工场景 |
| lithography machine semiconductor | 半导体设备大门类 + 光刻机载体 |
| PCB circuit board | 电子制造大门类 + 电路板载体 |

### 错误示例（禁止）

| 错误检索词 | 错误原因 |
|-----------|----------|
| 焊接芯片 | 太微观、太窄 |
| 拧螺丝 | 太微观、太窄 |
| 指纹解锁 | 太微观、太窄 |
| 台积电2024Q3涨价公告截图 | 太具体、搜不到 |

## TTS时长估算公式 (0e公式 — 男生解说声音 1.1倍速)

```
TTS时长 = (中文字时长 + 数字时长 + 英文缩写时长 + 标点停顿) ÷ 1.1
分配时长 = max(ceil(TTS时长 + 1秒缓冲), 4秒保底)
```

| 元素 | 基准速率 | 1.1倍速实际 | 示例 |
|------|----------|------------|------|
| 中文字 | 4.5字/秒 | 4.95字/秒 | "存储芯片短缺" 6字 = 1.21s |
| 数字1位 | 0.5s | 0.45s | "5" |
| 数字2位 | 0.8s | 0.73s | "40"、"18" |
| 数字3位 | 1.2s | 1.09s | "192"、"230" |
| 数字4位+ | 1.5s | 1.36s | "3000"、"22000" |
| 英文缩写 | 0.8s/组 | 0.73s/组 | "GPU"、"HBM"、"CoWoS"、"MLCC" |
| 标点停顿 | 0.15s/个 | 0.14s/个 | 逗号、句号、问号、分号 |
| 缓冲 | +1s | +1s | 开头结尾自然停顿 |
| 最低保底 | — | 4s | 低于4s的段落拉到4s，保证画面可消化 |
| **最终分配** | **ceil(以上÷1.1 + 1s)** | — | 向上取整到秒，不低于4s |

### 超时拆分规则

- 估算 > 15s：**必须拆分**，按子句边界切，每段6-12s
- 估算 10-15s：可不拆分，但分配时长必须 >= ceil(估算+1s)
- 子段继承原段素材配置，素材描述可微调

### 超时陷阱

| 陷阱 | 示例 | 额外耗时 |
|------|------|----------|
| 多组数字堆叠 | "从25%到35%,突破45%" | +2.4s |
| 英文缩写密集 | "GPU的HBM用CoWoS封装" | +2.4s |
| 长句无断点 | 50字+无逗号 | 11s+ |
| 数据+英文+数字 | "192GB的HBM供给被三家垄断" | +2.0s |
