# 制作流程（Phase 1-7）

## Phase 1：TTS语音合成

1. 将每镜**完整文案句**（不是子句）提取为TTS输入
2. 使用Windows SAPI**逐完整句**合成语音（整句合成，不拆）
3. 输出 `chunk_000.mp3` 到 `chunk_NNN.mp3`（N=镜数，不是子句数）
4. 记录每句实际时长（用于后续素材时长匹配）
5. **★ 用ffprobe实测每段TTS实际时长，与TTS时长公式估算的分配时长比对**
   - 实际 <= 分配 → OK
   - 实际 > 分配 → 调整分配时长，重新合成该段素材
   - **全部验证通过后才进入Phase 2**

```powershell
& python tts_script.py
# 使用 win32com.client.Dispatch('SAPI.SpVoice')
# Rate=1, Volume=100
# 输出wav再转mp3
# 每个完整句子调用一次TTS，不拆子句
```

**时长对齐规则：** 每镜素材时长匹配TTS实际时长，素材可略长0.2-0.5秒做缓冲，不能短于语音（会黑屏）。

### TTS技术限制

| 限制 | 说明 |
|------|------|
| 中文声音 | Windows SAPI仅有Huihui一个中文声音，无法更换 |
| 语速 | 可调（Rate参数，默认1） |
| 断句 | 必须按完整语义句合成，不可按标点拆碎 |

## Phase 2：素材采集与生成

按分镜方案的素材类型逐镜采集。**严格按分镜方案中的搜索关键词、备选方案、技术/插件、转场运镜执行。**

### 2a. 平台实拍素材（抖音→B站→YouTube，优先抖音）
按"平台素材采集详细流程"执行：搜索关键词 → Playwright打开平台 → 模拟人工浏览预筛选 → 播放确认 → yt-dlp下载（带cookies）→ 截取10-30秒流畅片段 → 裁剪清洗 → 缩放1080×768。

### 2b. 素材网站（Pexels / Pixabay / Mixkit）
按"素材网站采集流程"执行：搜索关键词 → curl_cffi 反爬绕过 → 标签排除 → 缩略图 AI 分析 → 下载 → ffprobe 验证 → 裁剪到1080×768。

详见 `references/material-collection.md`「素材网站采集流程」章节。

### 2c. AI生成视频（优先） / AI图片+运镜（备选）
**优先**：尝试用WorkRally或其他AI视频工具生成动态视频。
**备选**：当AI视频不可用时，用image_generate（MCP工具）生成图片 → 按转场运镜字段用FFmpeg zoompan做运镜。

### 2d. 数据可视化动画
文案涉及数据/数字/对比/概念比喻时，必须用Pillow/matplotlib生成动画。
不能用静态图或无关AI图替代。

## Phase 3：贴片与文案条制作

### 3a. 上贴片（384px）
暗红底#5a0510 + 斜线纹理 + 金色粗体标题（不斜体，太长换行缩小，确保完整可见，不放截断）。

### 3b. 文案条（384px）
暗红底#5a0510 + 斜线纹理 + 白字#fff thin 400 54px，居中显示当前句。
长句（>18字）必须拆成2-4个子句，按字数比例分配时间轮播。**字号固定54px不缩小**，子句超宽时换行排版而非缩字号。
每子句生成独立PNG，合成时按时间切换。

### 3c. 下贴片（384px）
暗红底#5a0510 + 斜线纹理，不放标题。

---

## Phase 4：分镜合成

对每镜，将素材 + 上贴片 + 文案条 + 下贴片垂直拼接：

```
┌──────────────┐
│   上贴片 384px│  ← 暗红+斜线纹理+金色粗体标题（不斜体，换行缩小）
│   素材 768px  │  ← 纯净素材/数据可视化动画
│   文案条 384px│  ← 暗红+斜线纹理+白字
│   下贴片 384px│  ← 暗红+斜线纹理（不放标题）
└──────────────┘
     1080×1920
```

- 素材画面缩放到1080×768
- 四层垂直拼接为1080×1920
- 每镜输出 `seg_NN/final.mp4`
- 长句合成：素材视频不变，TTS整句播放，文案条用FFmpeg drawtext按时段切换子句

```bash
# 长句合成示例：整句TTS + 字幕轮播
ffmpeg -y \
  -i material_video.mp4 \
  -i full_sentence_tts.mp3 \
  -filter_complex "
    [0:v]scale=1080:768[vm];
    drawtext=text='子句1':fontsize=54:fontcolor=white:
      x=(w-text_w)/2:y=(h-text_h)/2:
      enable='between(t,0,3)'[sub1];
    drawtext=text='子句2':fontsize=54:fontcolor=white:
      x=(w-text_w)/2:y=(h-text_h)/2:
      enable='between(t,3,6)'[sub2];
    [top][vm][sub1][sub2][bot]vstack=inputs=5[vo]
  " \
  -map "[vo]" -map 1:a \
  -c:v libx264 -c:a aac -shortest seg_NN.mp4
```

## Phase 5：全帧验证（强制，零容忍）

**最终视频每2秒截帧，全片至少110帧。**

```bash
for t in $(seq 0 2 $total_duration); do
    ffmpeg -y -ss $t -i final.mp4 -frames:v 1 "frame_$(printf '%03d' $t).jpg"
done
```

### 最终视频逐段验证项

| 验证项 | 方法 | 合格标准 |
|--------|------|---------|
| 标题区 | 每段截1帧 | 暗红底+金色粗体标题（不斜体），字号可见不溢出 |
| 标题斜体 | 每段截1帧 | 标题文字必须是斜体 |
| 标题自适应 | 长标题截1帧 | 自动缩小字号适应宽度，不溢出 |
| 素材区内容 | 每段截3帧（开头/中段/结尾） | 与文案匹配，无水印/字幕/人脸/K线/PPT |
| 文案条 | 每段截1帧 | 暗红底+白字，文字可读，只显示当前一句 |
| 文案逐句切换 | 逐段检查 | 字幕只显示当前一句，不是全部堆一起 |
| 长句分幕 | 检查长句段 | 长句已拆子句，素材不变但字幕轮播 |
| 下贴片 | 每段截1帧 | 暗红底+斜线纹理，无标题文字 |
| 贴片颜色 | 每段截1帧 | 统一暗红#5a0510，无色差，有斜线纹理 |
| AI生成素材 | 每段截3帧 | 有明显动态运镜效果，不是静止画面 |
| 音画同步 | 逐段检查 | 语音与字幕对应，时间点匹配 |

## Phase 6：最终合成

```bash
# 视频拼接
ffmpeg -y -f concat -safe 0 -i concat.txt -c copy final_video.mp4

# 音频拼接
ffmpeg -y -f concat -safe 0 -i audio_concat.txt -c copy full_audio.mp3

# 音视频合并
ffmpeg -y -i final_video.mp4 -i full_audio.mp3 -c:v copy -c:a aac -shortest final.mp4
```

## 用户可指定素材

用户可提供视频/图片/链接 + 镜号，直接替换对应分镜。收到后：
1. 下载/复制用户素材
2. 裁剪/缩放到1080×768
3. 检查是否符合红线规则
4. 替换该镜素材
5. 重新合成该镜 + 最终视频
6. 重新验证
